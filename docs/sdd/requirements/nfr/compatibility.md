# 互換性要件

> 非機能要件カテゴリ: 互換性

## 概要

ccresmonは社内の主要なLinux環境（特にWSL2）で動作しなければならない。外部ツールへの依存を排し、標準的なLinux環境で利用可能なコマンド・ファイルのみを使用する。

## 要件一覧

### NFR-CMP-001: Linux（WSL2含む）対応

**EARS記法**: Linux環境（WSL2を含む）において、システムは正常に動作しなければならない

**測定基準**:
- 指標: 対象プラットフォームでの動作可否
- 目標値: Linux（Ubuntu 22.04以降）およびWSL2で動作すること
- 測定方法: 各環境での実行テスト

**関連する機能要件**: REQ-001-001, REQ-001-002, REQ-002-001

**優先度**: 高

**検証方法**:
- [ ] Ubuntu 22.04上で全機能が動作することを確認
- [ ] WSL2（Ubuntu）上で全機能が動作することを確認

---

### NFR-CMP-002: 外部依存なし

**EARS記法**: システムは標準的なLinux環境にプリインストールされているコマンドのみで動作しなければならない

**測定基準**:
- 指標: 使用する外部コマンドのリスト
- 目標値: `free`, `cat`, `awk`, `bash` 等の標準コマンドのみ
- 測定方法: スクリプト内で使用するコマンドの確認

**関連する機能要件**: REQ-004-002

**優先度**: 高

**検証方法**:
- [ ] 最小構成のUbuntuで追加パッケージなしに動作することを確認

---

### NFR-CMP-003: procfsの利用

**EARS記法**: システムは `/proc/loadavg` および `/proc/diskstats` からリソース情報を取得しなければならない

**測定基準**:
- 指標: リソース情報取得元
- 目標値: procfsのみ（メモリは `free` コマンド経由でも可）
- 測定方法: スクリプトのコードレビュー

**関連する機能要件**: REQ-001-002, REQ-002-001

**優先度**: 高

**検証方法**:
- [ ] `/proc/loadavg` が利用可能な環境で正しくCPU負荷を取得できることを確認
- [ ] `/proc/diskstats` が利用可能な環境で正しくI/O情報を取得できることを確認
