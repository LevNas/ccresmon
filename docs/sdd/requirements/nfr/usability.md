# ユーザビリティ要件

> 非機能要件カテゴリ: ユーザビリティ

## 概要

ccresmonは社内チーム向けに配布されるため、インストール・設定・運用のすべてにおいて簡単であることが求められる。技術的知識が少ないユーザーでも導入できることを目指す。

## 要件一覧

### NFR-USA-001: インストール手順の簡潔さ

**EARS記法**: システムは3ステップ以内の手順でインストールと初期設定が完了できなければならない

**測定基準**:
- 指標: インストールに必要なステップ数
- 目標値: 3ステップ以内
- 測定方法: READMEの手順をカウント

**関連する機能要件**: REQ-004-001, REQ-004-002

**優先度**: 高

**検証方法**:
- [ ] READMEに記載の手順が3ステップ以内であることを確認
- [ ] 新規ユーザーが手順通りに設定できることを確認

---

### NFR-USA-002: ブロック/警告メッセージの明瞭性

**EARS記法**: ブロックまたは警告が発生した時、システムは原因（どのリソースが閾値超過か）と現在値と閾値を明示しなければならない

**測定基準**:
- 指標: メッセージに含まれる情報の完全性
- 目標値: 原因・現在値・閾値の3要素を含む
- 測定方法: 出力メッセージの目視確認

**関連する機能要件**: REQ-001-003, REQ-001-004, REQ-002-002

**優先度**: 高

**検証方法**:
- [ ] メモリ超過時のメッセージに使用率・閾値が含まれることを確認
- [ ] CPU超過時のメッセージに負荷値・閾値が含まれることを確認
- [ ] ディスクI/O超過時のメッセージに使用率・閾値が含まれることを確認

---

### NFR-USA-003: フックの有効/無効切り替え

**EARS記法**: システムは `settings.json` のフック設定の追加/削除のみで有効/無効を切り替えられなければならない

**測定基準**:
- 指標: 切り替えに必要な操作
- 目標値: settings.jsonの編集のみ
- 測定方法: 手順の確認

**関連する機能要件**: REQ-004-004

**優先度**: 中

**検証方法**:
- [ ] フック設定を削除した状態でccresmonが実行されないことを確認
- [ ] フック設定を追加した状態でccresmonが正常に動作することを確認
