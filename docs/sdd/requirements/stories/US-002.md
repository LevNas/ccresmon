# US-002: Bash実行時のディスクI/O監視

## ユーザーストーリー

**私は** 低スペックPCでClaude Codeを使用するユーザーとして
**〜したい** Bashコマンド実行前にディスクI/O負荷をチェックし、高負荷時に警告してほしい
**なぜなら** ディスクI/Oが逼迫している状態でビルドやテストを実行するとPCの応答性が著しく低下するから

## 受入基準（EARS記法）

### 機能要件

- **REQ-002-001**: PreToolUseフックでBash実行が検出された時、システムはディスクI/O負荷を取得しなければならない
- **REQ-002-002**: もしディスクI/O負荷が閾値を超過しているならば、システムは警告メッセージを出力しなければならない
- **REQ-002-003**: ディスクI/O負荷が閾値を超過している場合でも、システムはBash実行をブロックせず許可しなければならない
- **REQ-002-004**: 警告メッセージには、現在のディスクI/O負荷値と閾値を含めなければならない

## データ要件

- ディスクI/O負荷: `/proc/diskstats` から算出（2回サンプリングによるI/O使用率）

## 備考

- ディスクI/Oは警告のみでブロックしない設計とする（ブロックするとClaude Codeの操作性に大きく影響するため）
- フック対象のツール名は `Bash` とする
- 警告時の戻り値は `{"decision": "allow", "reason": "..."}` とする（ブロックではなく情報提供）
